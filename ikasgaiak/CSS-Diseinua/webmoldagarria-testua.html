Testua eta irudiak

Testua

Dispositibo desberdinetara egokitu behar dugun diseinuaren beste alderdi bat testua da. Letra mota bat tamaina finkoarekin deklaratzen dugunean (14 pixel, adibidez), testua beti tamaina horretan agertzen da, gailua edozein dela ere. Ordenagailu pertsonaletan ondo ikus daiteke, baina pantaila txikietan horrelako testu bat irakurtzea zaila izango da. Letraren tamaina gailura egokitzeko, letra neurri estandarra erabil dezakegu. Tamaina hori nabigatzaileak zehazten du, exekutatzen den gailuaren arabera eta erabiltzailearen lehentasunetara egokituta. Adibidez, ordenagailu pertsonaletan, iturriaren tamaina lehenetsia normalean 16 pixel izaten da. Gure orrialdeko testua irakurgarria dela ziurtatzeko, haren tamaina defini dezakegu balio horren arabera. CSSk honako neurketa-unitate hauek eskaintzen ditu helburu horrekin.

  • em — Unitate hau elementuari esleitutako iturriaren tamainari dagokion neurria da. Zenbaki hamartarrak onartzen ditu. 1 balioa iturriaren egungo tamainaren berdina da.
  • rem — Unitate honek erroko elementuari esleitutako iturriaren tamainari dagokion neurria adierazten du (normalean, < body> elementua). Zenbaki hamartarrak onartzen ditu. 1 balioa iturriaren egungo tamainaren berdina da.

Iturriak lehen bezala definitzen dira, baina px unitatearen ordez em unitatea jarri behar da, elementuari esleitutako egungo iturriaren tamainari dagokion iturriaren tamaina adierazteko. Adibidez, estilo-orri honek <h1> eta <p> elementuetarako letra-tamainak definitzen ditu gure dokumentuan em unitateak erabiliz (arauak 5-6 zerrendako dokumentua kontuan hartuta definitu dira).

        
* {
  margin: 0px;
  padding: 0px;
  }
  section {
  float: left;
  width: 61%;
  padding: 2%;
  margin-right: 5%;
  background-color: #999999;
  }
  aside {
  float: left;
  width: 26%;
  padding: 2%;
  background-color: #CCCCCC;
  }
  .berreskuratu {
  clear: both;
  }
  article h1 {
    font: bold 2em Georgia, sans-serif;
    }
    aside h1 {
    font: bold 1.2em Georgia, sans-serif;
    }
    aside p {
    font: 1em Georgia, sans-serif;
    }

em unitateak px unitatea ordezkatzen du, eta, beraz, nabigatzailearen ardura da testuaren tamaina pixeletan kalkulatzea, gaur egun elementuari esleitutako iturriaren tamainatik abiatuta. Balio hori kalkulatzeko, nabigatzaileak em zenbakia biderkatu egiten du iturri estandarraren egungo tamainarekin.

Adibidez, 5-14 zerrendako h1 artikulazio-arauak 2em tamainako <h1> elementua esleitzen du <article> elementuen barruan. Horrek esan nahi du idazmahaiko ordenagailu batean elementu horien testuak 32 pixeleko tamaina izango duela (16 * 2).

Ordeztu arauak dokumentu honetarako sortutako denamoldagarri.css fitxategian, 5-14 Zerrendaren kodearekin, eta ireki dokumentua zure nabigatzailean. Goiko irudiaren antzeko zerbait ikusi beharko zenuke.

em unitateak elementuari esleitutako iturriaren tamainari dagokion tamaina definitzen du. Horren arrazoia da elementuek edukiontzien propietateak heredatzen dituztela eta, beraz, haien letra-tamainak nabigatzaileak ezarritako estandarraz bestelakoak izan daitezkeela. Horrek esan nahi du elementu baten edukiontziaren letra tamaina aldatzen badugu, elementuari esleitutako letra neurriak eragina izango duela.

Adibidez, honako arau hauek letra-tamaina bat esleitzen diote <article> elementuari eta beste bat <h1> elementuari bere barruan.

        
article {
  font: bold 1.5em Georgia, sans-serif;
  }
  article h1 {
    font: bold 2em Georgia, sans-serif;
}

Lehenengo erregelak 1.5em-eko letra-tamaina ematen dio <article> elementuari. Propietate hori aplikatu ondoren, <article> elementuaren edukiak, barruan <h1> elementuaren edukia barne, tamaina estandarra halako 1,5 izango du (24 pixel ordenagailu pertsonal batean). <h1> elementuak balio hori heredatzen du, eta, beraz, hurrengo araua aplikatzen denean, letraren tamaina ez da kalkulatzen tamaina estandarretik, edukiontziak ezarritako tamainatik baizik (16* 1.5* 2 = 48). Emaitza 5-7 irudian erakusten da.

Portaera hori aldatu nahi badugu eta nabigatzailea letraren tamaina kalkulatzera behartu nahi badugu, betiere balio estandarretik abiatuta, rem unitateak erabil ditzakegu. Unitate horiek erroko elementuari buruzkoak dira, egungo elementuaren ordez, eta, beraz, balioak beti oinarrizko balio berarekin biderkatzen dira.

          
article {
  font: bold 1.5rem Georgia, sans-serif;
  }
  article h1 {
  font: bold 2rem Georgia, sans-serif;
  }

5-16 Zerrendako arauak 5-14 Zerrendako adibideari aplikatuz gero, nabigatzaileak iturriaren tamaina kalkulatzen du balio estandarretik abiatuta, eta artikuluaren izenburua tamaina horren bikoitzean erakusten du, 5-6 irudiak ilustratzen duenaren arabera, edukiontzirako adierazitako tamaina kontuan hartu gabe.

Oinarrizkoena:

portzentajeen ordez em unitateak erabil ditzake elementuen tamaina zehazteko. em unitateak erabiltzen dituenean, iturriaren tamainak zehazten du tamaina, edukiontziaren ordez. Teknika hau diseinu elastikoak sortzeko erabiltzen da. Informazio gehiago nahi izanez gero, bisitatu gure webgunea eta jarraitu kapitulu honetako estekak.

Frogak egiteko 'frogalekua'

Estekak froga simpleak egiteko aproposa den aplikazio bat irekiko dizu elementuek zer-nolako itxura hartzen duten ikusteko, baina hor egindako frogak komeni da kode editorera eramatea eta bertan egiaztatzea gure nahiari hurbiltzen zaion egindakoa.

Irudiak

Irudiak berez erakusten dira jatorrizko tamainan, eta horrek esan nahi du eskuragarri dagoen espaziora egokitzen direla, esplizituki adierazi ezean. Irudi finko bat irudi malgu bihurtzeko, haren tamaina ehunekotan adierazi behar dugu. Adibidez, hurrengo erregelak <img> elementuen zabalera konfiguratzen du gure dokumentuko <article> elementuen barruan, % 100eko balioarekin, eta, beraz, irudiek edukiontziaren zabalera bera izango dute.

          
article img {
  width: 100%;
  }

5-18 kode zatiko erregelak irudia hedatzen uzten du, jatorrizko tamaina hartu arte. Irudia edukiontzia bezain zabala izango da, edukiontziak irudiaren jatorrizko tamaina baino handiagoa izan ezean.

Egin ezazu zeuk:

erantsi 5-17 Zerrendaren araua aurreko adibiderako sortutako estilo orriari eta ireki dokumentua zure nabigatzailean. Irudia zabaldu edo zutabearekin batera uzkurtu beharko litzateke leihoaren tamaina aldatu ahala.

Irudiaren zabalerari ehuneko bat esleitzean, nabigatzailea irudiaren tamaina bere edukiontziaren tamainaren arabera kalkulatzera behartzen da (<article> elementua gure dokumentuan). Era berean, % 100etik beherako balioak ere adieraz ditzakegu, baina irudiaren tamaina edukiontziaren tamainarekiko proportzionala izango da beti, eta horrek esan nahi du edukiontzia hedatuz gero, irudia jatorrizkoa baino tamaina handiagoarekin ager litekeela. Irudia hedatzeko edo biltzeko mugak ezarri nahi baditugu, max-width eta min-width propietateak erabili behar ditugu. Ikusi ditugu ezaugarri horiek kutxa malguaren ereduan aplikatuta, baina kutxa-eredu tradizionalean ere erabil daitezke, elementu malguetarako mugak adierazteko. Adibidez, irudia bere jatorrizko tamainan erakusteko leku nahikorik ez dagoenean bakarrik murriztea nahi badugu, max-width propietatea erabil dezakegu.

          
article img {
  max-width: 100%;
  }

5-18 kode zatiko erregelak irudia hedatzen uzten du, jatorrizko tamaina hartu arte. Irudia edukiontzia bezain zabala izango da, edukiontziak irudiaren jatorrizko tamaina baino handiagoa izan ezean.

Egin ezazu zeuk

Ordeztu 5-17 Zerrendaren araua zure estilo orrian 5-18 Zerrendaren erregelarekin. Ireki dokumentua zure nabigatzailean eta aldatu leihoaren tamaina. Irudia jatorrizko tamainara iritsi arte zabaltzen ikusi beharko litzateke, 5-8 irudiak erakusten duen bezala.

Irudia eskuragarri dagoen espaziora egokitzeaz gain, webgune moldagarri batek irudi batzuk beste batzuekin ordezkatzea ere behar izaten du, baldintzak gehiegi aldatzen direnean. Gutxienez bi egoeratan da beharrezkoa hori: eskuragarri dagoen espazioa jatorrizko irudia erakusteko nahikoa ez denean (webgunearen logoa bertsio murriztu batez ordezkatu behar denean, adibidez) edo pantailaren pixelen dentsitatea desberdina denean eta bereizmen altuagoa edo baxuagoa duen irudi bat erakutsi behar dugunean. Arrazoia edozein dela ere, HTMLk elementu hau eskaintzen du erakutsi beharreko irudia hautatzeko.

  • <picture> — Elementu hori edukiontzi bat da, eta <img> elementu bererako iturri ugari zehazteko aukera ematen digu.
  • <source> — Elementu horrek <img> elementurako iturri posible bat definitzen du. Batez besteko atributua izan dezake, irudia zein Media Queryrekin lotuta egongo den zehazteko, eta srcset atributua elementuaren iturri gisa deklaratu nahi ditugun irudien ibilbidea zehazteko.

<picture> elementua <source> elementu bat edo gehiago dituen edukiontzi bat baino ez da, irudiaren iturri posibleak zehazteko, eta <img> elementu bat amaieran, hautatutako irudia pantailan erakusteko. Dokumentu honek erakusten du nola ordezkatu webgune baten logoa pantaila txikiak dituzten gailuen bertsio sinplifikatu batekin, elementu horiek erabiliz.

              
<!DOCTYPE html>
<html lang="es">
  <head>
    <title>Orriaren izena hemen doa</title>
    <meta charset="utf-8">
    <meta name="description" content="Dokumentu hau HTML5a da">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <meta name="viewport" content="width=device-width, initial-scale=1">
  </head>
  <body>
    <header>
      <picture>
        <source media="(max-width: 480px)" srcset="logotxikia.jpg">
        <img src="logo.jpg">
      </picture>
    </header>
  </body>
</html>

5-19 kode zatiko dokumentuan, irudirako iturri bakarra deklaratzen dugu, eta Media Query, max-width ekin lotzen dugu: 480px. Nabigatzaileak dokumentu hau irakurtzen duenean, lehenik <source> elementua prozesatzen du eta, ondoren, <img> elementuari iturria esleitzen dio, baldin eta bistaratze-eremuak Media Query-ren baldintzak betetzen baditu. Kointzidentziarik aurkitzen ez bada, <img> elementuaren src atributuak zehaztutako irudia agertuko da haren ordez. Horrek esan nahi du bistaratze eremua 480 pixeletik gorakoa den bakoitzean, logo.jpg irudia kargatu eta pantailan erakusten dela.

Bistaratze-eremuaren zabalera 480 pixelekoa edo txikiagoa bada, nabigatzaileak <source> elementuak zehaztutako irudia <img> elementuari esleitzen dio, eta logotxikia.jpg irudia pantailan erakusten da.

Egizu zeuk

Sortu HTML fitxategi berri bat 5-19 zerrendako dokumentuarekin. Deskargatu logo.jpg eta logotxikia.jpg irudiak gure webgunetik, eta mugitu dokumentua dagoen direktoriora. Ireki dokumentua zure nabigatzailean eta aldatu leihoaren tamaina. Egungo tamainaren arabera, 5-9 irudiaren edo 5-10 irudiaren antzeko zerbait ikusi beharko zenuke.

Frogak egiteko 'frogalekua'

Estekak froga simpleak egiteko aproposa den aplikazio bat irekiko dizu elementuek zer-nolako itxura hartzen duten ikusteko, baina hor egindako frogak komeni da kode editorera eramatea eta bertan egiaztatzea gure nahiari hurbiltzen zaion egindakoa.

Irudi bat tamaina edo eduki desberdineko beste batekin ordezkatu behar den egoerarik ohikoena hori den arren, bada beste baldintza bat gure erabiltzaileei irudiak erakusten dizkiegunean kontuan hartu behar duguna. 2010ean Applek erretina motako monitoreak sartu zituenetik, pixel dentsitate handiagoa duten pantailak daude. Dentsitate altuagoak hazbeteko pixel gehiago esan nahi du, eta horrek irudi argiagoak eta zehatzagoak ekartzen ditu. Baina ezaugarri hori aprobetxatu nahi badugu, bereizmen handiko irudiak eman behar ditugu pantaila horietan. Adibidez, tamaina bereko eremu baten barruan 300 pixeleko zabalera eta 200 pixeleko altuera duen irudi bat ondo ikusiko da dentsitate normala duten pantailetan, baina pixel dentsitate handia duten gailuek irudia luzatu beharko dute eremu berean eskuragarri dauden pixel guztiak okupatzeko. Irudi hori erretina pantailetan ere ondo ikustea nahi badugu, 2ko eskala batekin (pixel kopuru bikoitza areako), irudi berarekin ordezkatu behar dugu, bi aldiz bereizmen handiagoarekin (600 x 400 pixel).

Media Queries-ek “resolution” gako-hitzarekin ikusten dute egoera hori. Gako-hitz honek balioa zenbaki oso batekin eta dppx unitatearekin adieraztea eskatzen du. 1 balioak bereizmen estandarra adierazten du. Balio altuagoek pantailaren eskala definitzen dute.

Gaur egun, erretina pantailek 2 eta 3ko eskalak dituzte. Hurrengo dokumentuak pantailetarako iturri espezifiko bat adierazten du, 2ko eskala batekin (ohikoenak).

              
<!DOCTYPE html>
<html lang="eus">
  <head>
    <title>Hauxe da orriaren izena</title>
    <meta charset="utf-8">
    <meta name="description" content="Hau HTML5 dokumentua da">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="stylesheet" href="denamoldagarri.css">
  </head>
  <body>
    <section>
      <div>
        <article>
          <h1>Lehen artikuluiaren izenburua</h1>
          
          <picture>
            <source media="(resolution: 2dppx)" srcset="miimagen@2x.jpg">
            <img src="nireirudia1.jpg">
          </picture>
        
        </article>
      </div>
      <div>
        <article>
          <h1>Bigarren artikuluaren izenburua</h1>
          <picture>
            <source media="(resolution: 2dppx)" srcset="miimagen@2x.jpg">
            <img src="nireirudia2.jpg">
          </picture>
        </article>
      </div>
    </section>
    <aside>
      <div>
        <h1>Alboko informazioa</h1>
        <p>Lehen artikuluaren aipamena</p>
        <p>Bigarren artikuluaren aipamena</p>
      </div>
    </aside>
    <div class="berreskuratu"></div>
  </body>
</html>

Dokumentu hau pixel dentsitate estandarra duen pantaila batean irekitzen bada, nabigatzaileak miimagen.jpg irudia erakusten du, baina gailuak pantaila bat duenean, 2ko eskala batekin erretinatzen du, nabigatzaileak miimagen @ 2x.jpg irudia <img> elementuari esleitzen dio eta pantailan erakusten du.

5-20 kodeko adibideak erakusten duen bezala, prozesua erraza da. Bi irudi sortu behar ditugu, bata bereizmen normalarekin eta bestea bereizmen altuagoarekin erretina pantaila duten gailuetarako, eta, ondoren, eskura dauden iturriak <source> elementuekin adierazi. Baina honek arazo bat dauka. Irudien tamaina zehazten ez badugu, tamaina originalean erakusten dira eta, beraz, bereizmen handiko irudiaren tamaina irudi arruntaren bikoitza izango da. Horregatik, bereizmen baxuko eta handiko irudiekin lan egiten dugun bakoitzean, haien tamainak esplizituki aitortu behar ditugu. Adibidez, ondorengo arauek irudiaren zabalera konfiguratu egiten dute, edukiontziaren tamainaren % 100eko balioarekin, eta horrek bi irudiek tamaina bera hartzea eragingo du.
