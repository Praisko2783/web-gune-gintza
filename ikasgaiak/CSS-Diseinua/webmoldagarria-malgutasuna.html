Moldagarritasuna, Box-sizing, Finko-moldakor

Web diseinu moldagarri batean, gure orrialdeak malguak izatea ahalbidetu behar dugu, elementuak erabilgarri dagoen espaziora egokitzeko, bistaratze eremuaren zabalera etete puntuen artean dagoenean. Hori kutxa malguaren ereduarekin lor daiteke. Edukiontzi malguak sortu eta haien tamainak FLEX propietatearekin malgutzat jo besterik ez dugu egin behar (ikus 4. kapitulua), baina kutxa eredu hori ez dute nabigatzaile batzuek onartzen, eta, beraz, garatzaile gehienek kutxa eredu tradizionala ezartzen dute oraindik.

Kutxa-eredu tradizionala ez zen elementu malguekin lan egiteko diseinatu, baina malgu bihur ditzakegu elementuak, tamainak ehunekotan adierazita. Elementu baten tamaina ehunekotan adierazten dugunean, nabigatzaileak pixelen benetako tamaina kalkulatzen du, edukiontziaren tamainatik abiatuta. Adibidez, 1000 pixeleko tamaina duen beste elementu baten barruan dagoen elementu bati % 80ko zabalera esleitzen badiogu, elementuaren zabalera 800 pixelekoa izango da (1000koaren % 80). Edukiontzien tamainak aldatzen duen bakoitzean elementuen tamaina aldatu egiten denez, edukiontziak malguak bihurtzen dira.

Aurreko ataleko kodea hartuko dugu abiapuntu atal honetako adibideak osatzeko.

Goiko arauek <section> eta <aside> elementuen tamaina adierazten dute. Lehenari edukiontziaren tamainaren % 65 eskaintzen dio eta bigarrenari, <aside> elementuari edukiontziaren % 25. Pantailaren tamaina aldatzen bada, nabigatzaileak berriro kalkulatzen du elementuen tamaina, eta, beraz, elementu horiek espazioaren arabera hedatzen edo murrizten dira.

Elementuaren zabalera ehunekotan aitortzen dugun bakoitzean, balioen batura osoa % 100etik gorakoa ez dela ziurtatu behar dugu. Bestela, elementuak ez dira espazio erabilgarrian sartuko eta lerro berri batera mugituko dira. Goiko kodearen adibidean, erraza izan zen hori lortzea, marjinarik gabeko bi elementu baino ez genituelako, betelanak edo ertzak, baina betelana bezalako balio gehigarri bat gehitu bezain laster, gogorarazi behar dugu balio horiek elementuaren zabalerari kendu behar dizkiogula edo guztizkoak % 100 gaindituko duela.

Adibidez, honako arau hauek <section> eta <aside> elementuen zabalera murrizten dute, alde bakoitzean % 2ko betegarria eta erdian % 5eko marjina gehitu ahal izateko.

Estekak froga simpleak egiteko aproposa den aplikazio bat irekiko dizu elementuek zer-nolako itxura hartzen duten ikusteko, baina hor egindako frogak komeni da kode editorera eramatea eta bertan egiaztatzea gure nahiari hurbiltzen zaion egindakoa.

Box-sizing

Elementu batek hartzen duen eremuaren guztizkoa kalkulatzen den bakoitzean, nabigatzaileak amaierako balioa lortzen du, tamaina + marjinak + betegarria + ertzak formularekin. Width propietatea 100 pixeleko balioarekin aitortzen badugu, margina 20 pixelera, padding 10 pixelera eta ertzak pixel batera, elementuak okupatutako eremu horizontalaren guztizkoa 100 + 40 + 20 + 2 = 162 pixel izango da (margin, padding eta border propietateen balioak formulan bikoiztu egin dira, kutxaren ezkerreko eta eskuineko aldeei balio berberak esleitu zaizkiela onartzen dugulako).

Horrek esan nahi du width propietatea duen elementu baten tamaina adierazten dugun bakoitzean, gogoratu behar dugula elementua erakusteko orrian behar den eremua handiagoa izan daitekeela. Egoera hori bereziki problematikoa da tamainak ehunekotan definitzen direnean, edo balioak ehunekotan beste unitate mota batzuekin konbinatzen saiatzen garenean, hala nola pixelekin. CSSk honako propietate hau du portaera hori aldatzeko.

Besterik adierazi ezean, box-sizing jabetzaren balioa content-box gisa aitortzen da, eta horrek esan nahi du nabigatzaileak padding eta border propietateen balioak gehituko dizkiola width propietateak zehaztutako tamainari. Aldiz, propietate horri border-box balioa esleitzen badiogu, nabigatzaileak betegarria eta ertza sartzen ditu zabaleraren zati gisa, eta, orduan, beste aukera batzuk ditugu, hala nola portzentajean definitutako zabalerak pixeletan egindako betegarriekin konbinatzea, hurrengo adibidean erakusten den bezala.

            
* {
  margin: 0px;
  padding: 0px;
  }
  section {
  float: left;
  width: 65%;
  padding: 20px;
  margin-right: 5%;
  background-color: #999999;
  box-sizing: border-box;
  }

  float: left;
  width: 30%;
  padding: 20px;
  background-color: #CCCCCC;
  box-sizing: border-box;

  .recuperar {

  }

Goiko kodean box-sizing propietatea aitortzen dugu, bi elementuetarako border-box balioarekin, <section> eta <aside≫. Hori dela eta, jada ez dugu betegarria kontuan hartu behar elementuen tamaina kalkulatzen dugunean (65 + 5 + 30 = 100).

Finkoa eta malgua

Aurreko adibideetan betegarri finkoak dituzten kutxa malguak nola definitu ikusi dugu, baina askotan zutabe osoak balio malgu eta finkoekin konbinatu behar izaten ditugu.

Berriro ere, hori lortzea oso erraza da kutxa malguaren ereduarekin; edukiontzi malgu bat sortu besterik ez dugu egin behar, flex propietatearekin malguak izatea nahi dugun elementuen tamaina adierazi eta width propietatearekin tamaina finkoa izatea nahi dugunenak (ikus 4. kapitulua, 4-37 kodea).

Baina kutxa-eredu tradizionala ez zen horrelako zereginak egiteko diseinatu. Zorionez, badira helburu hori lortzeko ezar ditzakegun trikimailu batzuk. Alternatibarik ezagunena da betegarri bat deklaratzea zutabe malguan, neurri finkoko zutabea kokatzeko eta betegarriak utzitako espazio hutsera zutabe hori desplazatzen duen marjina negatibo bat gehitzeko. Honako hauek dira <section> elementurako zabalera malgua eta gure dokumentuaren <aside> elementurako zabalera finkoa adierazteko behar ditugun arauak.

            
* {
  margin: 0px;
  padding: 0px;
  }
  section {
  float: left;
  
  width: 100%;
  padding-right: 260px;
  margin-right: -240px;
  
  box-sizing: border-box;
  }
  section > div {
  padding: 20px;
  background-color: #999999;
  }
  aside {
  float: left;
  width: 240px;
  padding: 20px;
  background-color: #CCCCCC;
  box-sizing: border-box;
  }
  .berreskuratu {
  clear: both;
  }

goiburuan arauek % 100eko zabalera eta 260 pixeleko betegarria esleitzen diote <section> elementuari. Betegarria width propietatearen balioan sartzeko box-sizing propietatea erabiltzen dugunez, elementuaren edukiak ezkerreko aldea bakarrik hartuko du, eta espazio huts bat utziko dugu eskuinean, tamaina finkoko zutabea kokatzeko. Azkenik, <aside> elementuak sortutako zutabea espazio horretara mugitzeko, 240 pixeleko marjina negatiboa deklaratzen dugu <section>> elementuaren eskuinaldean.

Oinarrizkoena:

Adibide horretan bi zutabeak ezkerretara flotarazten ditugu, eta horrek esan nahi du bata bestearen ondoan lerro berean kokatuko direla. Horrelako kasuetan, espazio bat sor dezakegu zutabeen erdian, padding propietatearen balioa handituz. Adibidean 260 pixeleko betegarria eta 240 pixeleko marjina deklaratzen ditugu; horrek esan nahi du tamaina finkoko zutabea 240 pixelera baino ez dela mugituko. ezkerrekoa, 20 pixelen artean tartea utziz.

Betegarria erabiltzen ari garenez eskuineko zutabea kokatzeko espazio huts bat sortzeko, zutabearen betegarri normala eta hondoaren kolorea <div> elementu edukiontziari esleitu behar dizkiogu, <section> elementuaren barruan. Horri esker, zutabearen edukiari betegarri bat gehitzeaz gain, atzealdeko kolore bat esleitu ahal izango diogu edukiak hartzen duen eremuari soilik, pantailako bi zutabeak bereiziz.

Egin ezazu zeuk:

Ordeztu arauak zure denamoldagarri.css fitxategian goiko kodearekin. Ireki dokumentua zure nabigatzailean. Arrastatu leihoaren alde bat tamaina aldatzeko. Ezkerreko zutabea zabaldu edo uzkurtu egin beharko litzateke, eta eskuineko zutabea beti tamaina berekoa (240 pixel).

Kodearen gainerakoa aurreko adibidearen antzekoa da, baina oraingoan betelana eta ezkerreko aldeko <section> elementuaren marjina aldatu behar ditugu, eskuinekoaren ordez.

Oinarrizkoa:

Elementu bat posizio berri batera mugitzen dugunean, hurrengo elementuari marjina negatibo bat esleituz, nabigatzaileak lehen elementua bigarrenaren atzean aurkezten du, eta, beraz, lehen elementuak ez du gertaerarik jasotzen, hala nola erabiltzaileak esteketan egiten dituen klikak. Tamaina finkoko zutabea erabiltzailearentzat ikusgai eta eskuragarri dagoela ziurtatzeko, zutabe malguaren azpiko kodean adierazi behar dugu. Horregatik, <aside> elementuak sortutako zutabea ezkerrera mugitzeko ez dugu dokumentua aldatu behar izan. Zutabea orriaren barruan kokatzen da, float propietatearen bidez.

Mekanismo hori bera erabiliz, alboetan tamaina finkoko bi zutabe eta erdian zutabe malgu bat adierazi nahi baditugu, zutabeak edukiontzi baten barruan multzokatu behar ditugu, eta, ondoren, elementuei propietateak bi aldiz aplikatu, edukiontziaren barruan eta kanpoan. Hurrengo dokumentuan, <section> bi elementu multzokatuko ditugu <div> elementu baten barruan, edukiontzi izenarekin identifikatuta, ezkerreko eta erdiko zutabeak gordetzeko.
