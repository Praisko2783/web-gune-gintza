Elementu malguak

Edukiontzi malgu baten barruko elementu bat malgu bihurtzeko, halakotzat jo behar dugu. Honako hauek dira elementu malguak konfiguratzeko dauden propietateak. flex-grow — Propietate honek adierazten du elementua zein proportziotan zabalduko edo uzkurtuko den. Proportzioa zehazteko, kutxako gainerako elementuei (elementu anaiei) esleitutako balioak hartzen dira kontuan.

  • flex-shrink — Propietate honek adierazten du elementua zein proportziotan murriztuko den. Proportzioa zehazteko, kutxako gainerako elementuei (elementu anaiei) esleitutako balioak hartzen dira kontuan.
  • flex-basis — Propietate honek elementuaren hasierako tamaina adierazten du.
  • flex — Propietate honek aukera ematen digu flex-grow, flex-shrink eta flex-basis propietateen balioak aldi berean konfiguratzeko.

Kaxa malguak zabaldu edo uzkurtu egiten dira, aitaren kaxaren barruko espazio librea okupatzeko. Espazioaren banaketa gainerako kutxen propietateen araberakoa da. Kutxa guztiak malgu gisa konfiguratzen badira, bakoitzaren tamaina aita kutxaren tamainaren eta flex propietatearen balioaren araberakoa izango da.

          
            #kaxanagusia {
              display: flex;
              width: 600px;
              }
              #kaxanagusia > div {
              height: 145px;
              margin: 5px;
              background-color: #CCCCCC;
              }
              #kaxa-1 {
              flex: 1;
              }
              #kaxa-2 {
              flex: 1;
              }
              #kaxa-3 {
              flex: 1;
              }
              #kaxa-4 {
              flex: 1;
              }

Goiko adibidean, flex propietatearen flex-grow balioa baino ez dugu aitortzen, kutxak nola hedatuko diren zehazteko. Kutxa bakoitzaren tamaina kalkulatzeko, kaxanagusiaren tamainaren balioa flex propietatearen balioarekin biderkatzen da, kutxaren flex-growren balio guztien batura eginez. Adibidez, kutxa-1 elementurako formula (600 x 1/4 = 150) da. 600 balioa kaxanagusia-ren tamaina da, Kutxa 1 elementuari esleitutako flex-grow balioa da, eta Kutxa bakoitzari esleitutako flex propietatearen balioen batura da. Gure adibideko kutxa guztiek balio bera dutenez flex propietatean, kutxa bakoitzaren tamaina 150 pixelekoa izango da, marjinak kenduta (5 pixeleko marjina esleitu diogu <div> elementuari).

Goiko kodean, 2. balioa 1. kaxa-elementuaren flex propietateari esleitzen diogu. Orain, kutxa horren tamaina kalkulatzeko formula (600 x 2/5 = 240) da. kaxanagusia-ren tamaina aldatzen ez dugunez, formularen lehen balioa berdina da, baina bigarren balioa 2 da (1. kaxako flex propietatearen balio berria), eta balioen batura flex propietate guztietatik 5 da (2. kaxarako eta beste hiru kaxetako bakoitzerako). Gainerako kaxentzat formula bera erabiliz, tamaina hauek lor ditzakegu: (600 x 1/5 = 120).

Emaitzak alderatuta, espazioa nola banatzen den ikus dezakegu. Erabilgarri dagoen espazioa zatitan banatzen da, kutxa bakoitzaren flex propietatearen balioen baturaren arabera (gure adibidean, guztira, 5). Gero, zatiak kutxen artean banatzen dira. Kaxa-1 elementuak bi zati jasotzen ditu, flex propietatearen balioa 2 delako, eta gainerako elementuek zati bakarra jasotzen dute, flex propietateen balioa 1 delako.

Abantaila ez da bakarrik elementuak malgu bihurtzen direla, baizik eta elementu berri bat gehitzen dugun bakoitzean, ez dugula haren tamaina kalkulatu behar; kutxa guztien tamainak automatikoki birkalkulatzen dira. Ezaugarri horiek interesgarriak dira, baina badira beste agertoki batzuk ere. Adibidez, kutxetako bat zurruna denean eta tamaina esplizitua duenean, beste kutxak malgutu egiten dira, gainerako espazioa partekatzeko.

Tamaina esplizitua duen kutxa bat eduki dezakegun bezala, bi edo gehiago izan ditzakegu. Printzipioa bera da, baina geratzen den espazioa kutxa malguetan banatzen da. Balizko egoera batean, elementu baten tamaina aitortu beharko genuke, baina malgu mantendu. Konfigurazio hori lortzeko, flex (flex-shrink eta flex-basis) propietaterako erabilgarri dauden gainerako parametroak erabili behar ditugu.

              width: 600px;

              margin: 5px;
              background-color: #CCCCCC;
              }

              width: 300px;

              flex: 1;
              }

              flex: 1;

Goiko adibideko lehen kaxak 300 pixeleko tamaina du, eta, beraz, 300 pixeleko espazioa banatu behar da gainerako kaxen artean (600 - 300 = 300). Nabigatzaileak kutxa malgu bakoitzaren tamaina kalkulatzen du, lehen erabili dugun formula berarekin: 300 x 1/3 = 100

Oraingoan, hiru parametro aitortu dira kutxa bakoitzaren flex propietatearentzat. Kutxa guztien lehen parametroa (flex-grow) 1. balioarekin definitu da, eta hedapen-proportzio bera aitortu da. flex-shrink eta flex-basis parametroetarako balio erantsien arabera zehazten da diferentzia. flex-shrink parametroak flex-grow gisa lan egiten du, baina kutxak zer proportziotan murriztuko diren zehazten du, erabilgarri dagoen espazioan sartzeko. Gure adibidean, parametro honen balioa 1 da kutxa-1 elementuarentzat, eta 5 gainerako kutxentzat; horrek leku gehiago esleituko dio kutxa-1 elementuari. flex-basis parametroak, bestalde, hasierako balioa ezartzen dio elementuari. Beraz, flex-basis parametroaren balioa elementu malgu bat zenbat zabalduko edo murriztuko den kalkulatzeko hartzen da kontuan. Balio hori 0 denean edo deklaratzen ez denean, aintzat hartutako balioa elementuaren edukiaren tamaina da.

flex-grow edo flex-shrink parametroei esleitutako 0 balioak ez du ahalbidetzen elementua zabaltzea edo murriztea, hurrenez hurren. Malgutasuna adierazteko, parametro horien balioak 1en berdinak edo handiagoak izan behar dira. Kutxa bakoitzari esleitutako espazio-zatia kalkulatzeko, lehengo formula bera erabiliko dugu: 100 × 1/4 = 25. Horrek esan nahi du 25 pixel gehiago gehitzen zaizkiola kutxa bakoitzaren tamaina gustukoenari (marjinak kenduta).

flex-basis propietateak aukera ematen digu hasierako tamaina bat zehazteko, erabilgarri dagoen espazioa kutxen artean banatzerakoan

abigatzaileari eragiteko, baina kutxak balio horretatik haratago hedatzen edo murrizten dira oraindik. CSSk honako propietate hauek eskaintzen ditu kutxa baten tamainari mugak jartzeko.

  • max-width —Propietate honek elementurako baimendutako gehieneko zabalera zehazten du. Onartu balioak CSSn eskuragarri dauden edozein unitatetan, pixeletan edo ehunekoetan, esaterako.
  • min-width —Propietate honek elementurako baimendutako gutxieneko zabalera zehazten du. Onartu balioak CSSn eskuragarri dauden edozein unitatetan, pixeletan edo ehunekoetan, esaterako.
  • max-height —Propietate honek elementurako baimendutako gehienezko altuera zehazten du. Onartu balioak CSSn eskuragarri dauden edozein unitatetan, pixeletan edo ehunekoetan, esaterako.
  • min-height —Propietate honek elementurako baimendutako gutxieneko altuera zehazten du. Onartu balioak CSSn eskuragarri dauden edozein unitatetan, pixeletan edo ehunekoetan, esaterako.
