Ohiko kaxen eredua

Ohiko kaxen ereduari jarraikiz, block elementuak bata bestearen azpian jartzen dira eta inline elementuak ezkerretik eskuinera kokatzen dira lerro berean. ohiko kaxen ereduak ezartzen duenez, elementuek leihoaren alde banatan har dezakete lekua, eta lerro berean partekatu espazioa beste elementu batzuekin, edozein motatakoak direla ere.

Demagun gure diseinuan zutabeak irudikatzen dituzten bi block elementu ditugula, zutabe bat ezkerrean eta bestea eskuinean, elementuak nahi dugun aldera higiaraziz. Honako hauek dira CSSk zeregin horretarako eskaintzen dituen propietateak.

  • float —Propietate honek aukera ematen dio elementu bati alde batera edo bestera flotatzeko eta erabilgarri dagoen espazioa okupatzeko, baita linea beste elementu batekin partekatu behar duenean ere. Balio erabilgarriak none (flotatzen ez duen elementua), left (ezkerrerantz flotatzen duen elementua) eta right (eskuinerantz flotatzen duen elementua) dira.
  • clear —Propietate honek dokumentuaren fluxu normala berrezartzen du eta ez die uzten elementuei ez ezkerretara ez eskuinera desplazatzen. Balio erabilgarriak none, left, right eta both (biak) dira.

float propietatearen ondorioz, elementua alde batera edo bestera higitzen da erabilgarri dagoen espazioan. Propietate hori aplikatzen denean, elementuek ez diote dokumentuaren fluxu normalari jarraitzen, erabilgarri dagoen espazioaren ezkerrera edo eskuinera mugitzen dira, float propietatearen balioari erantzunez eta clear propietatearekin kontrakoa zehaztu ezean.

Eduki higikorra

float eta clear propietateak, jatorrian, edukia elementu baten inguruan flotarazteko erabili ziren. Adibidez, testu bat irudi baten ondoan erakustea nahi badugu, irudia ezkerrerantz edo eskuinerantz flotaraz dezakegu, eta testuak irudiarekin partekatuko du lerro berean eskuragarri dagoen espazioa.

        
        <!DOCTYPE html>
        <html lang="es">
          <head>
            <title>Hemen doa dokumentauren izenburuao</title>
            <meta charset="utf-8">
            <meta name="description" content="Hau HTML5 dokumentua da">
            <meta name="keywords" content="HTML, CSS, JavaScript">
            <link rel="stylesheet" href="enestiloak.css">
          </head>
          <body>
            <section>
              <img src="eneirudia.jpg" width="120">
              <p>HTMLk, HyperText Markup Language ingelesezko siglak (hipertestu-marken lengoaia), web-orriak egiteko markatze-lengoaiari egiten dio erreferentzia. Web lengoaiarik garrantzitsuena da, eta berebiziko garrantzia du World Wide Web (WWW) sortu, garatu eta hedatzean.</p>
            </section>
            <footer>
              <p>Wikipediak argitaratua</p>
            </footer>
          </body>
        </html>

Goiko dokumentuak irudi bat eta paragrafo bat ditu. Dokumentu hau irekitzen badugu lehenetsitako estiloak erabiliz, <p> elementua <img> elementuaren azpian agertuko da.

Testua irudiaren ondoan erakutsi nahi badugu, <img> elementua ezkerrerantz edo eskuinerantz flotatzea eragin dezakegu. 

            
              section img {
                float: left;
                margin: 0px 10px;
                }

Elementu batek alde batera flotatzen duenean, beste elementuek bere inguruan kokatzen dira, espazio erabilgarria okupatuz.

left balioaren ordez float propietateari right balioa esleitzen badiogu, irudiak eskuinerantz flotatuko du eta testuak ezkerreko aldean dagoen espazioa hartuko du. 

          
            section img {
              float: right;
              margin: 0px 10px;
              }
Egin ezazu zuk zeuk

Sortu HTML fitxategi berri bat goian daukazun dokumentuarekin eta CSS fitxategi bat izen bera eta 4-3 Zerrenda erregela dituena. Gogoratu miimagen.jpg irudia direktorio berean sartzea (irudia gure webgunean dago eskuragarri). Ireki dokumentua zure nabigatzailean. 4-4 irudiak erakusten duenaren antzeko zerbait ikusi beharko nuke. Esleitu right balioa float jabetzari eta eguneratu dokumentua zure nabigatzailean. Irudia eskuinerantz mugitu beharko litzateke, 4-5 irudiak erakusten duen bezala.

Nabigatzaileek ezin dute edukiontzi baten tamaina kalkulatu elementu flotatzaileen tamainatik abiatuta; beraz, float jabetzak eragindako elementua lerro bereko gainerako elementuak baino altuagoa bada, edukiontzia gainezkatuko du. Esate baterako, hurrengo irudiak azaltzen du zer gertatzen den <img> elementuaren width atributua gure adibidean ezabatzen badugu eta nabigatzaileari uzten badiogu irudia jatorrizko tamainan erakusten (adibide honetan, hondo gris bat esleitzen diogu <section> elementuari, okupatzen duen eremua identifikatu ahal izateko).

          
          (...)
              <section>
                <img src="eneirudia.jpg" width="-----">
                <p>HTMLk, HyperText Markup Language ingelesezko siglak (hipertestu-marken lengoaia), web-orriak egiteko markatze-lengoaiari egiten dio erreferentzia. Web lengoaiarik garrantzitsuena da, eta berebiziko garrantzia du World Wide Web (WWW) sortu, garatu eta hedatzean.</p>
              </section>
          (...)

Nabigatzaileak testuaren tamainaren arabera kalkulatzen du edukiontziaren tamaina, eta, beraz, irudia edukiontziaren mugetatik kanpo dago. Irudiak ezkerrerantz egiten duenez eta <section> elementutik kanpora hedatzen denez, hurrengo elementuaren edukiak flotatzen jarraitzen du irudiak utzitako espazioa okupatu arte, eta, horrela, 4-6 irudian erakutsitako emaitza lortzen dugu (<footer> elementua irudiaren eskuineko aldean agertzen da, irudiaren azpian egon beharrean). Edukiontzia ur gaineko elementuei eusteko bezain altua dela ziurtatzeko modu bat overflow propietatea esleitzea da, auto balioarekin. Horrek edukiontziaren tamaina kalkulatzera behartzen du nabigatzailea, barruan dauden elementu guztiak kontuan hartuta.

          
            section {
              background-color: #CCCCCC;
              overflow: auto;
              }
              section img {
              float: left;
              margin: 0px 10px;
              }

Overflow propietateak ez dio edukiontziko edukiari uzten edukiontziaren tamaina luzatuz edo desplazamendu-barrak sartuz eduki guztia ikusi ahal izateko, edukiontziaren tamaina ezin bada aldatu.

Egin ezazu zuk zeuk

Ezabatu <img> elementuaren width atributua HTML dokumentuan. Ordeztu arauak CSS artxiboan goian ikusi dugun kodearekin. Ireki dokumentua zure nabigatzailean. Goiko irudian daukazunaren antzeko zerbait ikusi beharko zenuke.

Dokumentuaren fluxua normalizatzeko beste aukera bat da edukiontziaren barruan elementu flotatzailearen jarraian dagoen elementuaren clearpropietatea deklaratzea. Elementu flotatzaile baten ondoren beti ez dugunez elementu anaiarik, elementuek flotatzen jarrai ez dezaten, teknika honek dokumentuari elementu gehigarri bat eranstea eskatzen du. Adibidez, <p> elementuaren azpian <div> elementu bat sar dezakegu gure dokumentuko <section> elementuaren barruan, hurrengo elementuek alboetarantz flotatzen jarrai ez dezaten.

        
      <DOCTYPE html>
        <html lang="es">
          <head>
            <title>Hemen doa dokumentuaren izenburua</title>
            <meta charset="utf-8">
            <meta name="description" content="Hau HTML5 dokumentua da.">
            <meta name="keywords" content="HTML, CSS, JavaScript">
            <link rel="stylesheet" href"enestiloak.css">
          </head>
          <body>
            <section>
              <img src="eneirudia.jpg">
              <p>HTML-k, HyperText Markup Language-ren ingelesezko siglak, hipertexto-marken lengoaiari egiten dio erreferentzia. Web-orriak egiteko markatzeko lengoaiari egiten dio erreferentzia. Web-lengoaiarik garrantzitsuena hartzen da kontuan, eta funtsezko asmakizuna da World Wide Webaren agerpenean, garapenean eta hedapenean.
              <div class="clearfloat"></div>
            </section>
          <footer>
            <p>Wikipediak argitaratuta</p>
          </footer>
        </body>
      </html>

Kasu honetan, clear propietatea elementu gehigarriari esleitu behar diogu. Propietateak alde bateko edo bietako fluxua leheneratzeko balioak eskaintzen ditu. Normalean dokumentuaren fluxu normala bi aldeetan berrezarri behar dugunez, both izaten da gogokoen dugun balioa.

          
          section {
            background-color: #CCCCCC;
            }
            section img {
            float: left;
            margin: 0px 10px;
            }
            .clearfloat {
            clear: both;
            }

Goiko arauekin,<div> elementuak dokumentuaren fluxu normala berrezartzen du, nabigatzaileari edukitik abiatuta edukiontziaren tamaina kalkulatzeko aukera emanez, beheko irudian ikusten dugun emaitza lortzeko.

Egin ezazu zuk zeuk

Eguneratu zure HTML fitxategia goian ikusitako dokumentuko kodea erabiliz eta ordeztu CSS artxiboko erregelak goiko kodearekin. Ireki dokumentua zure nabigatzailean. Azken irudiaren antzeko zerbait ikusi beharko nuke.

Frogak egiteko 'frogalekua'

Estekak froga simpleak egiteko aproposa den aplikazio bat irekiko dizu elementuek zer-nolako itxura hartzen duten ikusteko, baina hor egindako frogak komeni da kode editorera eramatea eta bertan egiaztatzea gure nahiari hurbiltzen zaion egindakoa.
