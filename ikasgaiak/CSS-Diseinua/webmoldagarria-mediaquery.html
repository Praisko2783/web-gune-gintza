Media Query-ak, Etenguneak, Bistaratze-eremua

2007. urtean iPhonea merkatuan sartzeak mundua betiko aldatu zuen. Ordura arte, pantaila txikiak zituzten mugikorrak baino ez genituen, webguneak deskargatzeko eta erakusteko gaitasun nahikorik ez zutenak. Webean nabigatu nahi bagenuen, ordenagailu pertsonal batean egin behar genuen. Garatzaileek erraz kontrolatzen zuten egoera hori.

Erabiltzaileek pantaila mota bakarra zuten, eta pantaila horrek behar zuten guztia sartzeko adinako tamaina zuen. Orduko ebazpenik ezagunenek 1024 pixel horizontal biltzen zituzten gutxienez, eta horrek aukera ematen zien garatzaileei beren webguneak tamaina estandarrarekin diseinatzeko (aurreko kapituluan aurkeztu genuena bezala). Baina iPhone merkatuan agertu zenean, horrelako diseinu finkoa duten webguneak ezin ziren hain pantaila txikian irakurri.

Lehenengo irtenbidea bi webgune bereizi garatzea izan zen, bata ordenagailuentzat eta bestea iPhoneentzat, baina gailu berriak merkatuan sartzeak, hala nola iPad-a 2010ean, berriro behartu zituen garatzaileak alternatiba hobeak bilatzera. Elementuei malgutasuna ematea izan zen inplementatutako irtenbideetako bat, baina ez zen nahikoa izan. Hainbat zutabe dituzten webguneak ez ziren ondo erakusten pantaila txikietan. Azken soluzioak elementu malguak izan behar zituen, diseinua gailu bakoitzera egokitzeko aukeraz gain. Horrela sortu zen gaur egun web diseinu moldagarri gisa ezagutzen duguna, edo ingelesezko responsive web design izenez ezagutzen duguna.

Web diseinu moldagarria diseinu malguak eta Media Queries (bitartekoak kontsultatzea) izeneko CSSz hornitutako tresna bat konbinatzen dituen teknika bat da, pantailaren tamaina detektatzeko eta diseinua egoera bakoitzera egokitzeko behar diren aldaketak egiteko aukera ematen diguna.

Media Query-ak

Media Query bat CSSn erreserbatutako arau bat da, eta garatzaileei dokumentua erakusten den bitartekoa detektatzeko aukera emateko txertatu zen.

Adibidez, Media Queries erabiliz, dokumentua monitore batean erakusten den edo inprimagailu batera bidaltzen den zehaztu dezakegu, eta kasu bakoitzerako estilo egokiak esleitu. Horretarako, Media Queries-ek gako-hitz hauek eskaintzen dituzte.

  • all — Propietateak bitarteko guztietan aplikatzeko erabiltzen da.
  • print — Propietateak web-orria inprimagailu batera bidaltzen denean aplikatzen dira.
  • screen — Propietateak web-orria koloretako pantaila batean agertzeko aplikatzen dira.
  • speech — Web-orria ahots-sintetizadore baten bidez prozesatzen denean aplikatzen dira propietateak.

Weba diseinatzeko Media Querie-ak erabilgarriak izatea eragiten duena da ingurunearen tamaina ere antzeman dezaketela. Media Queriesekin bistaratze eremuaren tamaina antzeman dezakegu (gure webguneak erakusten dituen nabigatzailearen leihoaren zatia), eta gailu bakoitzerako CSS arau desberdinak definitu. Ezaugarri horiek detektatzeko hainbat gako-hitz erabil ditzakegu. Honako hauek dira web diseinu moldagarria duen webgune bat garatzeko gehien erabiltzen direnak.

  • width — Gako-hitz horrek propietateak aplikatzen diren zabalera zehazten du.
  • height — Gako-hitz horrek zehazten du propietateak zein altueratan aplikatzen diren.
  • min-width — Gako-hitz horrek propietateak aplikatzeko gutxieneko zabalera zehazten du.
  • max-width — Gako-hitz horrek propietateak aplikatzeko gehieneko zabalera zehazten du.
  • aspect-ratio — Gako-hitz horrek zehazten du propietateak zer proportziotan aplikatzen diren.
  • orientation — Gako-hitz horrek zehazten du propietateak zer orientaziotan aplikatzen diren. Eskuragarri dauden balioak portrait (bertikala) eta landscape (horizontala) dira.
  • resolution — Hitz gako horrek zehazten du propietateak aplikatzen diren pixelen dentsitatea. Hazbeteko (dpi), zentimetroko (dpcm) edo pixeletako (dppx) proportzioko puntuetan balioak onartzen ditu. Adibidez, erretina motako pantaila bat 2 eskalan detektatzeko, 2dppx balioa erabil dezakegu.

Gako-hitz horiek erabiliz, orrialdea erakutsiko den ingurunearen eta bistaratze-arloaren zenbait alderdi antzeman ditzakegu, eta propietateak aldatu, diseinua egungo baldintzetara egokitzeko. Adibidez, Media Query width gakoarekin eta 768px balioarekin definitzen badugu, propietateak soilik aplikatuko dira orrialdea portrait moduan (orientazio bertikala) iPad estandar batean erakusten denean.

Media Query bat definitzeko, hitz gakoak eta behar ditugun balioak bakarrik adieraz ditzakegu. Ezaugarri bat deskribatzen duten gako-hitzak, hala nola bistaratze-eremuaren zabalera, parentesi artean adierazi behar dira. Gako-hitz bat baino gehiago sartuz gero, and (y) eta or (o) eragile logikoekin lotu ditzakegu. Adibidez, Media Query all and (max-width: 480px) propietateak esleitzen dizkio dokumentuari bitarteko guztietan, baina soilik bistaratze eremuak 480 pixeleko zabalera edo txikiagoa duenean.

Bi modu daude Media Queries adierazteko:

dokumentutik bertatik <link> elementuaren batez besteko atributua erabiliz.

estilo orritik@media erregela erabiliz

<link>

elementua erabiltzen dugunean, CSS fitxategia hauta dezakegu, konfigurazio espezifiko baterako kargatu nahi dugun estilo orriarekin. Adibidez, hurrengo dokumentuak bi CSS fitxategi kargatzen ditu, bata egoera eta bitarteko guztietan aplikatuko ditugun estilo orokorrak dituena, eta bestea pantaila txikiko gailu batean orrialdea aurkezteko behar diren estiloekin (480 pixel zabal edo gutxiago).

          
<!DOCTYPE html>
<html lang="es">
  <head>
    <title>Orriaren izena hemen</title>
    <meta charset="utf-8">
    <meta name="description" content="HTML5 dokumentu bat da hau">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <link rel="stylesheet" href="denamoldagarri.css">
    <link rel="stylesheet" media="(max-width: 480px)" href="sakelakoakmoldagarri.css">
  </head>
  <body>
    <section>
      <div>
        <article>
          <h1>Lehenengo artikuluaren izenburua</h1>
          <img src="lehenIrudia.jpg">
        </article>
        <article>
          <h1>Bigarren artikuluaren izenburua</h1>
          <img src="bigarrenIrudia.jpg">
        </article>
      </div>
    </section>
    <aside>
      <div>
        <h1>Alboko informazioa</h1>
        <p>Lehenengo artikuluari aipamena</p>
        <p>Bigarren artikuluari aipamena</p>
      </div>
    </aside>
    <div class="berreskuratu"></div>
  </body>
</html>

Nabigatzaileak dokumentu hori irakurtzen duenean, lehenik eta behin, denamoldagarri.css fitxategia kargatzen du, eta, gero, bistaratze-eremuaren zabalera 480 pixelekoa edo txikiagoa bada, fitxategia sakelakoakmoldagarri.css kargatzen du eta estiloak aplikatzen dizkio dokumentuari.

(Cascading Style Sheets) 'estilo orriak kaskadan' izenaren inizialak CSS dira, eta horrek esan nahi du propietateak ur-jauzian prozesatzen direla eta, beraz, propietate berriek aurrekoak ordezten dituztela. Bistaratze eremu jakin baterako elementu bat aldatu behar dugunean, aldatu nahi ditugun propietateak adierazi besterik ez dugu egin behar, baina aurretik definitutako gainerako propietateen balioak berdinak dira.

Adibidez, dokumentuaren gorputzari atzealdearen kolore bat eslei diezaiokegu denamoldagarri.css fitxategian, eta, gero, sakelakoakmoldagarri.css fitxategian alda dezakegu, baina balio berri hori dokumentua pantaila txiki batean aurkezten denean bakarrik aplikatuko da. Honako hau da fitxategian ezarriko dugun erregela denamoldagarri.css.

          
body {
  margin: 0px;
  padding: 0px;
  background-color: #990000;
  }

sakelakoakmoldagarri.css fitxategian definitutako estiloek aldatu nahi ditugun propietateen balioak bakarrik aldatu behar dituzte (kasu honetan background-color propietatea), baina gainerako balioak denamoldagarri.css fitxategian adierazi bezala mantenduko, haiek baitira lehenetsitakoak dira.

            
body {
  background-color: #3333FF;
  }
Egin ezazu zuk zeuk:

Sortu HTML fitxategi berri bat goiko dokumentuko kodearekin. Aldi berean sortu CSS fitxategi bat, denamoldagarri.css izenekoa, goiko koadroko CSS-arekin (margin: 0px; padding: 0px; background-color: #990000;); eta beste bat, sakelakoakmoldagarri.css izenekoa, beste koadroko CSS-arekin (background-color: #3333FF;).

Ireki dokumentua zure nabigatzailean. Orrialdea ikusi beharko zenuke hondo gorri batekin. Arrastatu leihoaren alde bat zabaleraren tamaina murrizteko.

Bistaratzeko eremuaren zabalera 480 pixelekoa edo txikiagoa denean, atzealdearen kolorea urdinera aldatzen ikusi behar duzu.

<link> elementuarekin eta batez besteko atributuarekin estilo orri desberdinak karga ditzakegu egoera bakoitzerako, baina propietate gutxi batzuk bakarrik aldatzen direnean, guztiak estilo orri berean sar ditzakegu eta Media Queries definitu @media erregelarekin, hurrengo adibideak erakusten duen bezala.

            
body {
  margin: 0px;
  padding: 0px;
  background-color: #990000;
  }

  @media (max-width: 480px) {
    body {
      background-color: #3333FF;
    }
  }

Media Query-a @media erregelarekin deklaratzen da, eta, ondoren, ingurunearen ezaugarriak eta giltzen arteko propietateak definitzen dituzten gako-hitzak eta balioak. Nabigatzaileak estilo-orri hori irakurtzen duenean, <body> elementuari body erregelaren propietateak esleitzen dizkio, eta, ondoren, bistaratze-eremuaren zabalera 480 pixelekoa edo txikiagoa bada, <body> elementuaren hondoko kolorea #3333FF (urdina) bihurtzen da.

Egin ezazu zuk zeuk:

Ordeztu arauak zure fitxategian, denamoldagarri.css, goiko azken kodearekin, eta ezabatu HTML dokumentuko bigarren <link> elementua (adibide honetan, zure estilo guztietarako CSS fitxategi bat baino ez duzu behar). Ireki dokumentua zure nabigatzailean eta murriztu leihoaren tamaina CSSk Media Queryk zehaztutako araua nola aplikatzen duen ikusteko, bistaratze-eremuaren zabalera 480 pixelekoa edo gutxiagokoa denean.

Frogak egiteko 'frogalekua'

Estekak froga simpleak egiteko aproposa den aplikazio bat irekiko dizu elementuek zer-nolako itxura hartzen duten ikusteko, baina hor egindako frogak komeni da kode editorera eramatea eta bertan egiaztatzea gure nahiari hurbiltzen zaion egindakoa.

Eten-puntuak

Aurreko adibidean, Media Query bat aitortzen dugu, bistaratze-eremuaren zabalera 480 pixelekoa edo txikiagoa den detektatzen duena, eta baldintza betez gero, propietateak aplikatzen dituena.

Hau praktika arrunta da. Egun hauetan merkatuan eskuragarri dauden gailuek tamaina desberdinetako pantaila ugari dituzte. Width gako-hitza erabiltzen badugu gailu espezifiko bat detektatzeko, ezagutzen ez ditugun edo merkatuan sartu berri diren gailu batzuk ez dira detektatuko eta gure webgunea modu desegokian erakutsiko dute.

Adibidez, Media Query width: 768px delakoak tamaina estandarreko iPadak bakarrik detektatzen ditu portrait moduan (bertikala), orientazio espezifiko horretako gailu horiek bakarrik dutelako tamaina hori. Landscape (horizontala) moduko tableta batek, edo bereizmen desberdina duen batek, edo 768 pixeleko pantaila txikiagoa edo handiagoa duen beste edozein gailuk, ez dute propietate horien eraginik izango, nahiz eta tamaina pixel bat edo bi bakarrik aldatu. Erroreak saihesteko, ez dugu tamaina espezifikorik hautatu behar. Horren ordez, diseinua nabarmen aldatzeko gehieneko edo gutxieneko puntuak ezarri behar ditugu, eta kutxa eredua erabili behar dugu ementuen tamaina erabilgarri dagoen espaziora egokitzeko, pantailaren zabalera puntu horien artean dagoenean. Horiek gehieneko eta gutxieneko puntuak max-width eta min-width gako-hitzekin zehazten dira, eta eten-puntuak (breakpoints ingelesez) esaten zaie.

Eten-puntuetan, web orri baten diseinuak aldaketa esanguratsuak eskatzen ditu pantailaren tamainara egokitu ahal izateko. Puntu horietan, elementuen malgutasuna ez da nahikoa kutxak erabilgarri dagoen espaziora egokitzeko, eta elementuak leku batetik bestera mugitu behar ditugu, edo diseinutik kanpo utzi, gure webgunea irakurgarria izaten jarrai dezan. Ez dago estandarrik ezarrita puntu horiek zehazteko; dena gure diseinuaren araberakoa da.

Adibidez, erabaki genezake bistaratze-eremuaren zabalera 480 pixelekoa edo txikiagoa denean zutabe bakarra erabili behar dugula informazioa aurkezteko, baina hori bakarrik egin daiteke jatorrizko diseinuak zutabe bat baino gehiago baditu. Balio ohikoenak 320, 480, 768 eta 1024 dira.

Etendura-puntu bat ezartzeko, max-width edo min-width gako-hitzak dituen Media Query bat deklaratu behar dugu; horrela, propietateak bistaratze-eremuaren tamainak muga horiek gainditzen dituenean aplikatuko dira. Erabiltzen dugun gako-hitza jarraitu nahi dugun norabidearen araberakoa da. Lehenik eta behin pantaila txikiko gailuetarako diseinatu nahi badugu, min-width duten gutxieneko tamainak ezarri behar ditugu, baina ordenagailu pertsonalen pantaila zabalei aplika dakiekeen diseinu orokor bat ezarri nahi badugu eta, ondoren, tamaina murriztu ahala propietateak aldatu nahi baditugu, hobe da maximoak max-width bidez ezartzea.

Adibidez, hurrengo estilo-orriak atzealdeko kolore lehenetsia esleitzen dio dokumentuaren gorputzari, baina pantailaren tamaina murrizten den heinean, kolorea aldatu egiten da.

      
body {
  margin: 0px;
  padding: 0px;
  background-color: #990000;
  }

  @media (max-width: 1024px) {
    body {
      background-color: #3333FF;
    }
  }
  @media (max-width: 768px) {
    body {
      background-color: #FF33FF;
    }
  }
  @media (max-width: 480px) {
    body {
      background-color: #339933;
    }
  }
  @media (max-width: 320px) {
    body {
      background-color: #CCCCCC;
    }
  }

Goiko estiloak HTML dokumentuari esleitzen zaizkionean, nabigatzaileak 1024 pixeletik gorako bistaratze-eremuan hondo gorria duen orria aurkezten du, baina kolorez aldatzen da zabalera Batezbesteko Kitek ezarritako mugen azpitik dagoen bakoitzean.

Egizu zuk zeuk:

Ordeztu denamoldagarri.css fitxategiko arauak goiko kodearekin. Ireki dokumentua zure nabigatzailean eta joan estutzen leihoaren zabalera. Bistaratzeko eremua 1024 pixeletik gorakoa bada, gorputza hondo gorri batekin erakusten da, baina tamaina 1024 pixelera edo gutxiagora murrizten bada, kolorea urdinera aldatzen da, 768 pixelera edo gutxiagora, arrosara aldatzen da, 480 pixelera edo gutxiagora berde bihurtzen da, eta 320 pixelera edo gutxiagora gris bihurtzen da.

Bistaratze-eremua

Bistaratze-eremua (viewport ingelesez) gure web orriak erakusten dituen nabigatzailearen leihoaren zatia da. Leihoaren eta bistaratze-eremuaren arteko erlazioa dela eta, biek tamaina bera izan beharko lukete gailu mugikorretan, baina hori ez da kasu guztietan betetzen. Lehenetsita, gailu batzuek 980 pixeleko zabalera esleitzen diote bistaratze-eremuari, tamaina erreala edo pantailaren benetako tamaina kontuan hartu gabe. Horrek esan nahi du gure estilo orrien Erdi Kitek 980 pixeleko zabalera izango dutela, bistaratze eremuaren tamaina guztiz desberdina denean.

Egoera hori normalizatzeko eta nabigatzailea pantailaren benetako tamainaren berdina den bistaratze-eremuaren tamaina definitzera behartzeko, <meta> elementua adierazi behar dugu gure dokumentuen goiburuan, viewport izenarekin eta ikusi nahi dugun zabalera eta eskala zehazten duten balioekin. Width eta initial-scale balioak behar dira bistaratze-eremuaren zabalera eta eskala adierazteko. Hurrengo adibideak <meta> elementua nola konfiguratu behar dugun erakusten du, nabigatzaileari gailuaren pantailaren benetako tamaina eta eskala definitzeko eskatzeko.

  
<!DOCTYPE html>
<html lang="es">
  <head>
    <title>Orriaren izena hemen</title>
    <meta charset="utf-8">
    <meta name="description" content="Hau HTML5 dokumentua da">
    <meta name="keywords" content="HTML, CSS, JavaScript">
    <meta name="viewport" content="width=device-width initial-scale=1">
    <link rel="stylesheet" href="denamoldagarri.css">
  </head>
  <body>
    <section>
      <div>
        <article>
          <h1>Lehen artikuluaren izenburua</h1>
          <img src="lehenIrudia.jpg">
        </article>
      </div>
      <div>
        <article>
          <h1>Bigarren artikuluaren izenburua</h1>
          <img src="bigarrenIrudia.jpg">
        </article>
      </div>
    </section>
    <aside>
      <div>
        <h1>Alboko informazioa</h1>
        <p>Lehen artikularen aipamena</p>
        <p>Bigarren artikuluaren aipamena</p>
      </div>
    </aside>
    <div class="berrreskuratu"></div>
  </body>
</html>

Oinarrizkoa:

Bistaratze-eremua deklaratzen duen <meta> elementuak beste balio batzuk izan ditzake atributuak konfiguratzeko, hala nola eskala minimoak eta maximoak (minimum-scale eta maximum-scale), edo erabiltzaileei web orria zabaltzen edo murrizten utzi nahi badiegu (user-scalable).

Informazio gehiago nahi izanez gero, bisitatu gure webgunea eta jarraitu kapitulu honetako estekak.
