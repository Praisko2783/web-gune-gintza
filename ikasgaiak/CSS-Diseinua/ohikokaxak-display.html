Kaxak (display)

Aurreko kapituluan aipatu dugun bezala, nabigatzaileek kaxa birtual bat sortzen dute elementu bakoitzaren inguruan, okupatzen duten eremua zehazteko. Kaxa horiek pantailan antolatzeko, elementuak oinarrizko bi motatan sailkatzen dira: Block (blokea) eta Inline (lerroan).

Bi mota horien arteko desberdintasun nagusia da block elementuek tamaina pertsonalizatua dutela eta lerro-jauziak sortzen dituztela; inline elementuek, berriz, edukiaren araberako tamaina dute, eta ez dute lerro-jauzirik sortzen. Beren ezaugarriak direla eta, block elementuak banan-banan jartzen dira lerro desberdinetan, eta inline elementuak bata bestearen ondoan jartzen dira lerro berean, espazio nahikorik ez badago behintzat eskuragarri, beheko irudian ikusten dugun bezala.

Beren ezaugarriak direla eta, block elementuak egokiak dira web orri batean zutabeak eta sekzioak sortzeko, eta inline elementuak, berriz, egokiak dira edukia irudikatzeko. Horregatik, dokumentu baten egitura definitzen duten elementuak, hala nola <section>, <nav>, <header>, <footer>, edo <div>, block elementu bezala deklaratzen dira lehenetsita. Bste batzuk <span>, <strong> edo <em> adibidez inline elementu gisa deklaratzen dira lehenetsita

.

Display

Elementu bat block edo inline motakoa izatea nabigatzaileak zehazten du, baina CSStik baldintza hori alda dezakegu display propietatearekin.

display propietateak elementua pantailan aurkezteko erabiltzen den kaxa mota definitzen du. Hainbat balio daude propietate horrentzat. Gehien erabiltzen direnak none (elementua ezabatzeko), block (elementua lerro berri batean eta tamaina pertsonalizatuarekin erakusteko), inline (elementua lerro berean erakusteko) eta inlineblock (elementua lerro berean eta tamaina pertsonalizatuarekin erakusteko).

Egiturazko elementuak block balioarekin konfiguratzen dira lehenetsita, eta edukia irudikatzen duten elementuak, berriz, inline gisa konfiguratu ohi dira. Elementu mota aldatu nahi badugu, display propietatea balio berri batekin esleitu besterik ez dugu egin behar. Horrela, HTML5ek gehitutako elementuak ez dituzten nabigatzaileen bertsio zaharrak behartu egingo ditugu. Elementu horiek nabigatzaile guztietan block elementu gisa interpretatzen direla ziurtatu nahi badugu, beheko erregela deklara dezakegu gure estilo-orrietan.

          
            header, section, main, footer, aside, nav, article, figure, figcaption
            {
            display: block;
            }

display propietateak beste balio batzuk ditu, block eta inline balioez gain. Adibidez, none balioak elementua ezkutatzen du. Balio hori elementu bati esleitzen zaionean, elementua ez balego bezala aurkezten da dokumentua. Erabilgarria da dokumentua dinamikoki JavaScript-etik aldatu nahi dugunean edo web diseinu moldagarria erabiltzen dugunean gure webgunea diseinatzeko, hurrengo kapituluetan ikusiko dugun bezala.

Oinarrizkoa

Propietatearen none balioak dokumentuaren elementua ezabatzen du. Elementu ikusezin hori agertu nahi badugu, beste CSS propietate bat erabil dezakegu, visibility izenekoa. Propietate horrek visibility eta hidden balioak onartzen ditu. Balioa propietateari esleitzen zaionean, elementua eta edukia sortzen dira (pantailan espazio bat hartzen dute), baina ez zaizkio erabiltzaileari erakusten.

display propietatearentzat beste balio erabilgarri bat inline-block da. block elementuek bi ezaugarri dituzte:

Batetik, lerro-jauzi bat eragiten dute; beraz, hurrengo elementua lerro berri batean agertzen da, eta, bestetik, tamaina pertsonalizatua har dezakete.

Horregatik, dagoeneko ikusi ditugun width eta height propietateek lehen block elementuekin bakarrik lan egiten zuten. Propietate horiek <span> bezalako inline elementu bati esleitzen badizkiogu, ez da ezer gertatzen. Baina display propietateak inline-block balioa eskaintzen du tamaina pertsonalizatua har dezakeen inline elementu bat definitzeko.

>Horri esker inline-Block elementuak bata bestearen ondoan jarriko dira ilara berean, baina nahi dugun tamainarekin.

inline-block elementuei esker, nahi dugun tamainako sekzioak sor ditzakegu gure webgunean, eta, behar izanez gero, lerro berean kokatu. Adibidez, bata bestearen ondoan erakutsi beharreko bi block elementu baditugu, hala nola 2. kapituluan garatutako dokumentuko <section> eta <aside> elementuak, inline-block elementu gisa deklara ditzakegu.

Gure webguneetako zutabe eta atal guztiak diseinatzeko orduan inline-block elementuak erabiltzea tentagarria izan daitekeen arren, CSSk propietate hobeak ditu helburu horretarako. Ezaugarri horiek kaxen eredua deitzen dugunaren zati bat dira, kaxak pantailan nola erakutsiko diren, betetzen duten espazioa eta eskuragarri dagoen espazioa kontuan hartuta orrian nola antolatzen diren zehazten duten arauen multzo bat.

Gaur egun hainbat kaxa-eredu daude eskuragarri, kaxa tradizionalaren eredua eta kaxa malguaren eredua estandartzat hartuta. Gure web orriak diseinatzen ikasteko, bi eredu horiek nola funtzionatzen duten ulertu behar dugu.

Frogak egiteko 'frogalekua'

Estekak froga simpleak egiteko aproposa den aplikazio bat irekiko dizu elementuek zer-nolako itxura hartzen duten ikusteko, baina hor egindako frogak komeni da kode editorera eramatea eta bertan egiaztatzea gure nahiari hurbiltzen zaion egindakoa.
