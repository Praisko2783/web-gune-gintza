CSS propietateak (animazioak)

transition propietateak oinarrizko animazio bat sortzen du, baina prozesuan bi egoera baino ez dira sartzen: propietateen egungo balioek zehaztutako hasierako egoera eta balio berriek zehaztutako azken egoera. Benetako animazio bat sortzeko, bi egoera baino gehiago aitortu behar ditugu, film baten fotogramak bezala. CSSk honako propietate hauek ditu animazio konplexuagoak osatzeko.

  • animation-name —Propietate honek animazioaren urratsak identifikatzeko erabiltzen den izena zehazten du. Hainbat animazio aldi berean konfiguratzeko erabil daiteke, izenak komaz bereizita adieraziz.
  • animation-duration —Propietate honek animazioaren ziklo bakoitzaren iraupena zehazten du. Balioa segundotan zehaztu behar da (adibidez, 1s = segundo bat).
  • animation-timing-function —Propietate honek zehazten du nola egingo den animazio-prozesua ease, linear, ease-in, ease-out eta ease-in-out balioekin deklaratutako Bézier kurba baten bidez.
  • animation-delay —Propietate honek zehazten du nabigatzaileak zenbat denbora itxarongo duen animazioa hasi aurretik.
  • animation-iteration-count —Propietate honek adierazten du zenbat aldiz exekutatuko den animazioa. Zenbaki oso bat edo balio infinitua onartzen du, eta, horren ondorioz, animazioa denbora mugagabez exekutatzen da. Lehenetsitako balioa 1 da.
  • animation-direction —Propietate honek animazioaren zuzendaritza adierazten du. Lau balio onartzen ditu: normal (lehenetsita), lehenestea, txandakatzea eta txandakatzea. Birgertatzearen balioak animazioaren norabidea alderantzikatzen du, eta urratsak adierazi diren kontrako norabidean erakusten ditu. Txandakako balioak animazioaren zikloak nahasten ditu, indize bakoitia norabide normalean dutenak eta gainerakoa norabide alderantzikatuan dutenak erreproduzituz. Azkenik, txandakako balioak –reverse– txandakatzen den gauza bera egiten du, baina alderantzizko zentzuan.
  • animation-fill-mode —Propietate honek definitzen du nola eragiten dien animazioak elementuaren estiloei. none (lehenetsita), forwards, backwards eta both balioak onartzen ditu. Forwards balioak animazioaren azken urratsean aplikatutako propietateek definitutako estiloekin mantentzen du elementua; backwards balioak, berriz, lehen urratseko estiloak aplikatzen ditu animazioa definitu bezain laster (exekutatu aurretik). Azkenik, both balioak bi efektuak sortzen ditu.
  • animation — Propietate honek animazioaren balio guztiak aldi berean definitzea ahalbidetzen digu.

Propietate horiek animazioa konfiguratzen dute, baina urratsak@keyframes arauaren bidez adierazten dira. Arau hori animazioa konfiguratzeko erabilitako izenarekin identifikatu behar da, eta urrats bakoitzean aldatu nahi ditugun propietateen zerrenda jaso behar du. Animazioaren urrats bakoitzaren posizioa ehunekotan zehazten da. % 0ko balioa lehenengo fotogramari edo animazioaren hasierari dagokio, eta % 100 amaierari..

          
          header {
            margin: 30px;
            padding: 15px;
            text-align: center;
            border: 1px solid;
            animation: eneanimazioa 1s ease-in-out 0s infinite normal none;
            }
            @keyframes eneanimazioa {
            0% {
            background: #FFFFFF;
            }
            50% {
            background: #FF0000;
            }
            100% {
            background: #FFFFFF;
            }
            }
            #titulua {
            font: bold 26px Verdana, sans-serif;
            }

Goikoko erregelek burualdeko hondoaren koloreak gorritik zurira aldatzen dituen animazio bat sortzen dute. Animation propietatearen bidez definitu da animazioa, segundo bateko iraupenarekin, eta balio infinituarekin behin eta berriz exekutatzeko konfiguratu da. Propietateak animazio izena ere ematen dio animazioari, ondoren @keyframes arauarekin konfiguratu ahal izateko.

Urrats bakoitzean, elementuari nola eragingo dion adierazten dute propietateek. Kasu honetan, hiru urrats aitortzen ditugu,

  • bat % 0an
  • beste bat % 50ean
  • hirugarren bat % 100ean

Animazioa hasten denean, nabigatzaileak % 0an definitutako estiloak esleitzen dizkio elementuari, eta, ondoren, propietateen balioak pixkanaka aldatzen ditu, % 50ean definitutako balioetara iritsi arte. Prozesua balio horretatik jabetzari azken urratsean esleitutako azken balioraino (% 100) errepikatzen da.

Adibide honetan, animazioaren hasierako egoera % 0an eta amaierako egoera % 100ean definitzen dugu, baina animazioa beste edozein puntutan ere has dezakegu eta behar ditugun pauso guztiak eman ditzakegu, hurrengo adibidean bezala.

            
              header {
                margin: 30px;
                padding: 15px;
                text-align: center;
                border: 1px solid;
                animation: eneanimazioa 1s ease-in-out 0s infinite normal none;
                }
                @keyframes eneanimazioa {
                20% {
                background: #FFFFFF;
                }
                35% {
                transform: scale(0.5);
                background: #FFFF00;
                }
                50% {
                transform: scale(1.5);
                background: #FF0000;
                }
                65% {
                transform: scale(0.5);
                background: #FFFF00;
                }
                80% {
                background: #FFFFFF;
                }
                }
                #titulua {
                font: bold 26px Verdana, sans-serif;
                }

Goiko kodean, animazioa % 20an hasi eta % 80an amaitzen da, eta guztira bost urrats hartzen ditu. Animazioaren urrats bakoitzak bi propietate aldatzen ditu, elementuaren tamaina handitu eta hondoaren kolorea aldatzen dutenak, lehen urratsa eta azkena izan ezik, horiek kolorea aldatzen baitute errebote-efektua lortzeko.

Egin ezazu zuk zeuk

Idatzi zure .css artxiboan goiko arauak eta ireki dokumentua zure nabigatzailean eta ikusi efektua. Jarraian frogatu atal honetan ikasitako funtzioak blur() funtzioa beste edozein funtzio erabilgarrirekin zer-nolako efektuak lor daitezkeen ikusteko.
